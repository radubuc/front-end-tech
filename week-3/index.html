Alpha sizing on column class doesn't refer to size of column, it refers to size of device screen! Column options: auto, sm, md, lg xl, xxl. No xs option because that is the default.

Quote from BOOTSTRAP GRID video: "If we have 3 extra large columns, they will remain side by side as long as the viewport, or screen, is 1140 px or larger, and as soon as the viewport is smaller, the columns will stack on top of each other. If we use 3 sm columns, the columns will remain side by side even on smaller viewports, as long as they are gte to a width of 576 px."

1. Template should span the entire width of the screen.

2. There should be three columns. The left and right column should each take up 25% the width of the screen when the viewport is anything larger than small. The middle column should take up the remaining 50% of the screen.

3. Inside the middle column there should be two more columns, the left column taking up 75% of the width of it's parent and the right column taking up the remaining 25% when the viewport is anything larger than small.
